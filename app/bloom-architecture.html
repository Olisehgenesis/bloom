4. Data Flow Summary

  • Identity: wallet address is the canonical user ID. SIWE proves ownership; Supabase session binds it to an HttpOnly cookie. Optional GoodDollar face verification adds proof-of-personhood.
  • Value: UGX → token (Fonbnk) → G$ (Uniswap V3 inside BloomProxy) → continuous stream (Superfluid) → recipient → off-ramp to MoMo (Fonbnk).
  • State: all financial state lives on-chain (deposits, swaps, streams). Supabase stores only the wallet ↔ session mapping. No PII is persisted server-side.
  • Audit: every deposit, swap, stream and claim emits a Celo event — publicly queryable, immutable, and exportable to M&E dashboards.

5. Security & Compliance Controls

ConcernControl
AuthenticationSIWE (EIP-4361). No passwords. Session cookies are HttpOnly, Secure, SameSite=Lax.
AuthorizationSupabase Row-Level Security; on-chain access controls on BloomProxy admin functions (Ownable + UUPS).
TransportTLS 1.3 end-to-end (Vercel + Supabase). HSTS enabled.
At-restSupabase encrypts disk volumes; secrets in Vercel encrypted env store.
Key custodySelf-custodial: user keys never touch Bloom servers.
Sybil resistanceGoodDollar face verification per recipient — eliminates ghost beneficiaries.
Audit logOn-chain events (immutable) + Vercel request logs (90 days).
Data protectionAligned with Uganda Data Protection & Privacy Act 2019: minimal PII, EU data region, user-controlled wallet identity.

6. Interoperability & Government Integration Points

  • NIRA / National ID: NIN can be bound to a wallet via a one-time attestation API (KYC tier upgrade).
  • URA: on-chain disbursement records exportable as CSV / API for tax-compliant reporting.
  • NITA-U: deployable behind NITA-U gateway; supports REST/JSON for inter-agency integration.
  • Mobile money: already live with MTN MoMo and Airtel Money via Fonbnk.
